[OpenTelemetry](https://opentelemetry.io/) is a collection of APIs, SDKs, and tools. Use it to instrument, generate, collect, and export telemetry data (metrics, logs, and traces) to help you analyze your software’s performance and behavior.

Using OpenTelemetry makes collecting logs, metrics, and traces a breeze.
> - Configure OpenTelemetry [settings](https://opentelemetry.io/docs/collector/configuration/) and define receiver, exporter, and connector like [here](https://github.com/newtuple/fantastic-fiesta/blob/main/OTEL/otel-collector-config.yaml).
> - Start the OpenTelemetry collector using [Docker](https://github.com/newtuple/fantastic-fiesta/blob/main/docker-compose-services.yml).
> - Configure [logger](https://github.com/newtuple/fantastic-fiesta/blob/main/bot_service/src/backend/common/logger.py) for OpenTelemetry to send log data into the collector, which finally exports it to the respective destination.
> - Comment out `logger_provider.shutdown()` to turn on the OpenTelemetry log ingestion.
