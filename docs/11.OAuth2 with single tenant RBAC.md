# Custom OAuth2 Authentication with single tenant RBAC

This document provides a comprehensive guide on setting up custom OAuth2 authentication with single tenant Role-Based Access Control (RBAC) for your application. It covers the necessary configurations, steps to implement OAuth2, and how to manage user roles effectively.


## Getting started

To start using the [custom OAuth2 authentication with single tenant RBAC which can easily be extended for multi-tenant scenarios / other OAuth2 providers](../bot_service/src/backend/auth/manager.py), follow these steps:

1. All the configurations are in the `configuration.py` file located at `bot_service/src/backend/configuration.py`. Update the following parameters as per your requirements:

```python
    "secret_key": os.environ.get("CUSTOM_OAUTH_SECRET_KEY",),
    "algorithm": os.environ.get("CUSTOM_OAUTH_ALGORITHM", "HS256"),
    "access_token_expire_seconds": int(os.environ.get("CUSTOM_OAUTH_ACCESS_TOKEN_EXPIRE_SECONDS", 3600)),
    "refresh_token_expire_seconds": int(os.environ.get("CUSTOM_OAUTH_REFRESH_TOKEN_EXPIRE_SECONDS", 86400)),
    "enable_bootstrap_admin": int(os.environ.get("CUSTOM_OAUTH_BOOTSTRAP_ADMIN", 1)),
    "bootstrap_admin_email": os.environ.get("CUSTOM_OAUTH_BOOTSTRAP_ADMIN_EMAIL"),
    "bootstrap_admin_password": os.environ.get("CUSTOM_OAUTH_BOOTSTRAP_ADMIN_PASSWORD"),
```

2. The bootstrap admin user is created only if the `enable_bootstrap_admin` flag is set to `1`. Ensure that you provide a valid email and password for the bootstrap admin user, which is the initial user with admin privileges.


3. After updating the configuration, add dependencies to the required endpoints to enforce authentication and role-based access control. For example, to protect an endpoint and restrict access to users with the "admin" role, you can use the following code snippet:

```python

class UserRestController:
    """Implements health REST controller"""

        self.current_db = database_service_manager.postgres_db_service()
        self.current_db_engine = self.current_db.engine
        self.auth_service_manager = auth_service_manager
        self.custom_oauth2_service = self.auth_service_manager.get_custom_oauth_service()
        self.redis_cache_config = self.auth_service_manager.config.configuration().redis_configuration

    def prepare(self, app: APIRouter) -> None:
        """
        Prepare the user REST controller.
        """
.

def get_current_user_with_roles(roles: list[str] | None = None, scopes: list[str] | None = None):
    def dep(current_user: Annotated[User, Security(self.custom_oauth2_service.get_current_user, scopes=scopes)]):
        if not current_user.is_active:
            raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail="Inactive user")
        if roles and current_user.role not in roles:
            raise HTTPException(status_code=status.HTTP_403_FORBIDDEN, detail="Forbidden")
        return current_user

    return dep

    @app.post(
        "/roles",
        status_code=status.HTTP_200_OK,
        tags=["roles", "admin"],
        response_model=RoleResponse,
    )
    def create_role(
        request: Request,
        role_request: RoleCreateRequest,
        auth_user: Annotated[User, Depends(get_current_user_with_roles(roles=["admin"], scopes=[]))],  # noqa: F841, ASYNC124
    ):
```

4. Use the `get_current_user_with_roles` dependency in your route handlers to enforce role-based access control. In the example above, only users with the "admin" role can access the `/roles` endpoint. Similarly, you can specify different scopes for granular access control on other endpoints as needed.

5. Email service is required to send password reset emails. Configure the email settings in the `configuration.py` file located at `bot_service/src/backend/common/configuration.py`. Update the following parameters as per your requirements:

```python
    "email_configuration": {
        "smtp_server": os.environ.get("SMTP_SERVER", "smtp.example.com"),
        "smtp_port": int(os.environ.get("SMTP_PORT", 587)),
        "smtp_username": os.environ.get("SMTP_USERNAME", "your_username"),
        "smtp_password": os.environ.get("SMTP_PASSWORD", "your_password"),
        "use_tls": os.environ.get("SMTP_USE_TLS", "true").lower() in ("true", "1", "yes"),
        "from_email": os.environ.get("FROM_EMAIL", "no-reply@example.com"),
        "from_name": os.environ.get("FROM_NAME", "Your App Name"),
    },
```
6. Ensure that your SMTP server is correctly configured and accessible from your application environment to enable email functionalities.
The proxy URL for the email service should also be configured if your application is running behind a reverse proxy.

```python
            "server_configuration": {

```
